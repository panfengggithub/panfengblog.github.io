    动态连通性：union-find算法

    panfeng 12/15/2020 算法

    # 动态连通性：union-find算法

    # 1. 什么是动态连通性？

    动态连通性，简单的说就是输入一列整数对，其中每个整数都表示某种类型的对象，整数对p和q可以被理解为“p和q是相连的”。

    假设相连是一种等价关系，这种等价关系一般具有三种特性：

      • 自反性：p和p是相连的。
      • 对称性：如果p和q是相连的，那么q和p也是相连的。
      • 传递性：如果p和q是相连的且q和r是相连的，那么p和r也是相连的。

    等价关系能够将对象分为多个等价类。在这里，当且仅当两个对象相连时它们才属于同一个等价类。

    (在下面的算法叙述中，我们将对象称为触点，整数对称为连接，将等价类称为连通分量或简称分量，)。

    简单的举个例子，计算机网络中判断两个计算机是否需要建立新的连接通信，如果可以通过一个或某几个节点能通信，那么我们就不需要建立新的连接，否则我们就union建立连接，使线路连通可以使用网络进行通信。

    通过上面的叙述，我们对动态连通性相关理论和概念已经有了一定的了解，如果还不是很了解，看看接下来的案例，相信你一定可以理解的更深透。

    接下来，我们开始分析和设计算法来解决动态连通性问题。

    # 2. 动态连通性需要解决什么问题？

    首先我们需要知道动态连通性需要解决什么问题，动态连通性主要解决以下两个问题：

      • 对于某个给定触点，能够查到它所在的分量是什么。
      • 当读取到两个不同分量里的对象时，可以将两个分量合并连通。

    接下来，我们通过一个案例的学习，来解决这两个问题。

    # 3. 动态连通性案例描述

    **我们的目标是编写一个程序来过滤掉序列中所有无意义的连接（**整数对称为连接，有意义连接：两个触点均来自于同一个分量中），将无意义连接中的触点输出打印，然后将无意义连接设置为某个分量。

    换句话说，当程序从输入中读取了整数对p和q时，如果已知的**所有整数对(所有连接)**都不能说明p和q是相连的，那么则将这一对整数输出打印，并设置为某个分量。如果已知的数据可以说明p和q是相连的，那么程序忽略p、q这对整数并继续处理输入中的下一对整数。

    问题描述我们已经了解了，接下来我们开始设计数据结构和算法来解决案例所描述的问题。

    # 4. 三种算法的共同API

    下面我们会使用三种算法来解决上述案例中的问题，每种算法都会有以下相同的API，但是具体的实现细节会有所不同。

    # 5. 算法一：quick-find

    以下算法用一个以触点为索引的数组作为数据结构来表示所有分量(int[] id)，初始时每个触点都构成一个只含有自己的分量。

    保证当且仅当 id[p] 等于 id[q] 时 p 和 q 是连通的。即在同一个分量中的所有触点i在 id[i] 中的值必须全部相同。

    如果p和q触点不在一个分量中，则合并这两个分量 , 并将id[p]所在的分量内的所有触点的分量值设置为id[q]。

    public class QuickFindUF {
    private int[] id;    // 用一个以触点为索引的数组作为数据结构来表示所有分量
    private int count;   // 分量的个数

    /**
     * 初始化数据，每个触点都构成一个只含有自己的分量
     */
    public QuickFindUF(int n) {
        count = n;
        id = new int[n];
        for (int i = 0; i < n; i++)
            id[i] = i;
    }

    /**
     * 返回分量的个数
     */
    public int count() {
        return count;
    }

    /**
     * 返回某个触点的分量值
     */
    public int find(int p) {
        validate(p);
        return id[p];
    }

    // 验证p是否为有效索引
    private void validate(int p) {
        int n = id.length;
        if (p < 0 || p >= n) {
            throw new IllegalArgumentException("index " + p + " is not between 0 and " + (n-1));
        }
    }

    /**
     * 如果两个元素在同一分量中，则返回true。
     */
    @Deprecated
    public boolean connected(int p, int q) {
        validate(p);
        validate(q);
        return id[p] == id[q];
    }

    /**
     * 合并
     */
    public void union(int p, int q) {
        validate(p);
        validate(q);

        //查询p,q触点的分量值
        int pID = find(p);
        int qID = find(q);

        // 如果p和q触点在一个分量中，则不采取任何操作
        if (pID == qID) return;

        //如果p和q触点不在一个分量中，则合并这两个分量，
        // 并将两个分量内的所有触点的分量值设置为qID
        for (int i = 0; i < id.length; i++)
            if (id[i] == pID) id[i] = qID;
        //合并分量后，分量数肯定减一
        count--;
    }

    /**
     * 输入一列整数进行动态连通性测试
     */
    public static void main(String[] args) {
        int n = StdIn.readInt();
        QuickFindUF uf = new QuickFindUF(n);
        while (!StdIn.isEmpty()) {
            int p = StdIn.readInt();
            int q = StdIn.readInt();
            //如果p和q在同一个分量中，则不采取任何操作，并跳过
            if (uf.find(p) == uf.find(q)) continue;
            //如果p和q触点不在一个分量中，则合并这两个分量，并输出打印这两个分量
            uf.union(p, q);
            StdOut.println(p + " " + q);
        }
        StdOut.println(uf.count() + " components");
    }

}

    算法分析：

    • 构造函数初始化数组中的每一个元素——O(N)
    • 每次find(）调用只需要访问数组一次——O(1)
    • 每次connect() 操作调用2次find()，即访问数组两次——O(1)
    • 每次union()算法访问数组次数：

    2（两次find获得分量值）+N（检查N个数是否有相同的分量值）+1（最少改变1次） = N + 3——O(N)

    2（两次find获得分量值）+N（检查N个数是否有相同的分量值）+N-1（最多改变N-1次）=2*N+1——O(N)

    显然，要让所有连通，则需要N-1次union，即至少(N-1)*(n+3)次访问数组，所以quick-find是O(N^2)。

    # 6. 算法二：quick-union

    quick-union相对与quick-find，速度会快一点，但是不能保证在所有情况下，quick-union会比算法一快。quick-union算法主要优化了quick-find的union方法。

    quick-union也是使用数组来实现，每个触点i所对应的 id[i] 元素都是同一个分量中的另一个父触点的名称（也可能是他自己）。

    在实现 find() 方法时，从给定触点i开始，由它链接得到另一个父触点id[i]，再由这个父触点id[i]的链接到达第三个父触点id[id[i]]，如此继续直到到达一个根触****点，即根触点指向根触点自己的链接。

    用节点(带触点名称的圆圈)表示触点，用从一个节点到另一个节点的箭头表示链接。

    p q那一行代表的数字0-9是所有触点的名称，他们下面对应的数字是他们各自的父触点的名称。

    public class QuickUnionUF{
    private int[] id;  // 链接数组，每一个触点i对应的父触点为id[i]
    private int count;     // 分量的个数

    /**
     * Initializes an empty union-find data structure with
     */
    public QuickUnionUF(int n) {
        id = new int[n];
        count = n;
        for (int i = 0; i < n; i++) {
            id[i] = i;
        }
    }

    /**
     * Returns the number of sets.
     */
    public int count() {
        return count;
    }

    /**
     * 返回p触点的根触点名称
     */
    public int find(int p) {
        validate(p);
        while (p != id[p])
            p = id[p];
        return p;
    }

    // validate that p is a valid index
    private void validate(int p) {
        int n = id.length;
        if (p < 0 || p >= n) {
            throw new IllegalArgumentException("index " + p + " is not between 0 and " + (n-1));
        }
    }

    /**
     * 如果两个元素在同一集合根节点下，则返回true
     */
    @Deprecated
    public boolean connected(int p, int q) {
        return find(p) == find(q);
    }

    /**
     *将两个分量进行合并
     */
    public void union(int p, int q) {
        int rootP = find(p);
        int rootQ = find(q);
        if (rootP == rootQ) return;
        id[rootP] = rootQ;
        count--;
    }

    /**
     * 输入一列整数进行动态连通性测试
     */
    public static void main(String[] args) {
        int n = StdIn.readInt();
        QuickUnionUF uf = new QuickUnionUF(n);
        while (!StdIn.isEmpty()) {
            int p = StdIn.readInt();
            int q = StdIn.readInt();
            if (uf.find(p) == uf.find(q)) continue;
            uf.union(p, q);
            StdOut.println(p + " " + q);
        }
        StdOut.println(uf.count() + " components");
    }
}

    最坏情况算法分析：

    • 构造函数初始化数组中的每一个元素——O(N)
    • 每次find()操作需要访问数组1~n-1次——O(1)~O(N)
    • 每次connect() 操作调用两次find()，需要访问数组2~2(n-1)次——O(1)~O(N)
    • 每次union()操作调用两次find()，需要访问数组2~2(n-1)次——O(1)~O(N)

    显然，要让所有连通，则需要N-1次union，即至少(N-1)*(1~N)次访问数组，所以quick-union是O(1)~O(N^2)，相对来说可能比quick-find快，但是极端情况可能两者差不多！

    # 7. 算法三：Weighted quick-union

    这种算法对quick-union进行改进，速度比之前两种算法都要快，我们称为加权quick-union算法，这种算法会记录每一棵树的大小并总是将较小的树连接到较大的树上。这里的树的大小，指的是一个树节点的数目，一个树的节点数目为1，则树大小为1，一个树的节点数目为2，则树大小为2，依次类推。

    我们可以用下图对比一下这两种算法进行动态链接后，树的高度对比，树的高度越低，find()从低到高查找根触点，查找的次数就越少，反之则越多。

    要想实现加权quick-union算法，我们需要添加一个记录所有树大小的数组size[]。其中find()方法不变，改变union()的算法，当rootP的树小于rootQ的树时，将rootP的父触点设置为rootQ，即将小的树的根触点指向较大的树的根触点，并将rootQ的树大小加上rootP树的大小，完成合并操作。

    public class WeightedQuickUnionUF {
    private int[] parent;   // parent[i] = parent 链接数组，每一个触点i对应的父触点为parent[i]
    private int[] size;     // size[i] = number 以i为根触点的的子树中元素的个数
    private int count;      // 分量的个数

    /**
     * 初始化每一个触点i对应的父触点数组parent
     * 每一个触点刚开始都是根触点，子树为自己，元素个数为1
     */
    public WeightedQuickUnionUF(int n) {
        count = n;
        parent = new int[n];
        size = new int[n];
        for (int i = 0; i < n; i++) {
            parent[i] = i;
            size[i] = 1;
        }
    }

    /**
     * Returns the number of sets.
     */
    public int count() {
        return count;
    }

    /**
     *  返回p触点的根触点名称
     */
    public int find(int p) {
        validate(p);
        while (p != parent[p])
            p = parent[p];
        return p;
    }

    /**
     * 如果两个元素在同一个集合中，则返回true。
     */
    @Deprecated
    public boolean connected(int p, int q) {
        return find(p) == find(q);
    }

    // validate that p is a valid index
    private void validate(int p) {
        int n = parent.length;
        if (p < 0 || p >= n) {
            throw new IllegalArgumentException("index " + p + " is not between 0 and " + (n-1));
        }
    }

    /**
     * 两个分量进行合并
     */
    public void union(int p, int q) {
        //找到p触点的根触点
        int rootP = find(p);
        //找到q触点的根触点
        int rootQ = find(q);
        if (rootP == rootQ) return;
        System.out.print("rootP="+rootP+
                " rootQ="+rootQ+" --- ");
        // 使较小的树的根触点指向较大的树的根触点
        if (size[rootP] < size[rootQ]) {
            parent[rootP] = rootQ;
            size[rootQ] += size[rootP];
        }
        else {
            parent[rootQ] = rootP;
            size[rootP] += size[rootQ];
        }
        count--;
    }


    /**
     * 输入一列整数进行动态连通性测试
     */
    public static void main(String[] args) {
        int n = StdIn.readInt();
        WeightedQuickUnionUF uf = new WeightedQuickUnionUF(n);
        System.out.println("---parent-init---"+Arrays.toString(uf.parent));
        System.out.println("----size-init----"+Arrays.toString(uf.size));
        while (!StdIn.isEmpty()) {
            int p = StdIn.readInt();
            int q = StdIn.readInt();
            if (uf.find(p) == uf.find(q)) continue;
            uf.union(p, q);
            StdOut.println(p + " " + q);
            System.out.println("---parent--------"+Arrays.toString(uf.parent));
            System.out.println("----size---------"+Arrays.toString(uf.size));
        }
        StdOut.println(uf.count() + " components");
    }
}

    因为加权的find-union有点复杂，这里我们需要验证一个命题，以便于下面的算法分析更好懂。

    **命题：**对于N个触点的树，加权quick-union算法构造的森林中的任意节点的高度(高度是从树的底层往上数)最多为lgN。

    证明：我们可以用归纳法证明。我们知道，当树大小k等于1时树的高度为0(这里我们把最底层设为0层，有的教材可能将最底层设为1层)。根据归纳法，我们假设大小为i的树的高度最多为lgi，其中i<k。设i≤j且i+j=k****,当我们将大小为i和大小为j的树归并时，quick-union 算法和加权quick-union算法中触点与深度如下图所示。**小树中的所有节点的深度增加了1,但它们现在所在的树的大小为i+j=k,而1+lgi=lg(i+i)≤lg(i+j)=lgk**,性质成立。

    上面是《算法》一书中的证明，看起来很难让人理解，但是我们可以采用曲线救国的方式来证明，我们可以证明：N个节点的堆的最大高度为lgN。

    首先说明一下，**堆是除根节点与最底层外，每一层节点都是2的倍数的树，**堆是一种特殊的完全二叉树。

    要证明N个节点的堆的最大高度为lgN，首先我们得知道在高度为h的堆中，最多和最少的节点个数是多少。详细计算证明步骤如下：

    最多：底层全满；2^0 + 2^1 + 2^2 + ...... + 2^h，等比数列求和得2^(h+1) - 1

    最少：底层只有一个节点；2^0 + 2^1 + 2^2 + ...... + 2^(h-1) + 1，等比数列求和得2^h - 1 + 1 = 2^h 我们假设N个元素的堆的高度为h。由上面堆节点个数范围得2^h <= N <= 2^(h+1) - 1<2^(H+1)，因此h <= lgN < h+1。

    由上面的quick-union算法图解我们可以知道，我们的算法形成的树和堆很像，而且除根节点和最底层外，每一层的节点数都大于二，所以树的高度肯定比堆的lgN要低。

    算法分析：

    • 构造函数初始化两个数组中的每一个元素2N次——O(N)
    • 每次find()操作需要访问数组lgN次，上述命题已证树的高度最多为lgN——O(lgN)
    • 每次connect() 操作调用两次find()，需要访问数组2lgN次——O(lgN)
    • 每次union()操作调用两次find()，需要访问数组2lgN次——O(lgN)

    显然，要让所有连通，则需要N-1次union，即至少(N-1)*2lgN次访问数组，所以Weighted quick-union是O(NlgN)，这是三种算法中速度最快的一种了，一般都能在常数级别完成每个操作。

    # 8. 三种算法对比

    我们通过测试以下网格中的所有节点(x,y)之间的连通性，来对我们上面我们所提到的三种算法进行测试性能，每一个节点坐标(x,y)对应一对整数对，我们主要是通过统计三种算法中相应数组访问的次数来对比。

    下面的坐标轴主要参数说明：

    • X轴代表900个整数对的输入次数
    • Y轴代表三种算法主要方法union()或find()访问相应数组的次数

    我们可以看出性能比较为：quick-find < quick-union < weighted quick-union
